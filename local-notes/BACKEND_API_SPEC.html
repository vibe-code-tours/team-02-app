Backend API Specification

v2.0 — Staff-Generated OTP

📋 Overview

SonicSavor is a restaurant web app with:

  • Staff-generated OTP authentication (no email required)
  • Admin panel for staff to manage bookings and generate OTPs
  • Table booking system (22 tables, 5 types)
  • Menu ordering with mood-based recommendations
  • Session management (4-hour guest sessions)

🔄 Authentication Flow (Updated)

Key Change: Staff generates OTP, NOT email-based. Customer receives code from staff (on screen).

🚶 Walk-in Customer

Customer arrives → Staff generates OTP → Staff shows code on screen → Customer enters code → Access granted

📅 Online Booking

Customer booked online → Staff clicks Check In → System generates OTP → Staff shows code → Customer enters code

👨‍💼 1. Admin Panel API (Staff)

POST /api/admin/generate-otp

Purpose: Staff generates OTP for walk-in customer

Request:

{
  "tableType": "private",
  "partySize": 4,
  "guestName": "Walk-in Guest"
}

Response (201):

{
  "success": true,
  "otp": {
    "code": "472891",
    "tableId": "P2",
    "tableType": "private",
    "expiresAt": "2026-07-14T18:00:00Z"
  },
  "message": "Show this code to customer"
}

POST /api/admin/check-in

Purpose: Check in online booking and generate OTP

Request:

{
  "bookingId": "BK-20260714-P1"
}

Response (200):

{
  "success": true,
  "otp": {
    "code": "839201",
    "tableId": "P1",
    "booking": {
      "guestName": "Sarah Chen",
      "partySize": 4,
      "occasion": "date_night"
    }
  }
}

GET /api/admin/bookings

Purpose: View today's bookings

Response (200):

{
  "date": "2026-07-14",
  "bookings": [
    {
      "id": "BK-20260714-P1",

GET /api/admin/sessions

Purpose: View active sessions

Response (200):

{
  "sessions": [
    {

      "guestName": "Walk-in Guest",

POST /api/admin/revoke

Purpose: Revoke a session

Request:

{
  "sessionToken": "abc123xyz",
  "reason": "Customer left"
}

🔐 2. Customer Authentication

POST /api/auth/verify-otp

Purpose: Customer enters OTP code from staff

Request:

{
  "code": "472891"
}

Response (200):

{
  "success": true,
  "sessionToken": "abc123xyz",
  "session": {
    "tableId": "P2",
    "tableType": "private",
    "guestName": "Walk-in Guest",
    "expiresAt": "2026-07-14T21:00:00Z"
  }
}
Note: No email required. Just enter the code staff gave you.

GET /api/auth/session

Purpose: Check current session

Response (200):

{
  "authenticated": true,
  "session": {
    "tableId": "P2",
    "tableType": "private",
    "guestName": "Sarah Chen",
    "preferences": {
      "occasion": "date_night",
      "dietaryRestrictions": ["vegetarian"]
    }
  }
}

📅 3. Online Booking (Customer)

POST /api/bookings

Purpose: Customer books table online

Request:

{
  "tableType": "private",
  "date": "2026-07-14",
  "time": "19:00",
  "partySize": 4,
  "guestName": "Sarah Chen",
  "guestEmail": "sarah@example.com",
  "occasion": "date_night",
  "specialRequests": "Anniversary dinner"
}

Response (201):

{
  "success": true,
  "booking": {
    "id": "BK-20260714-P1",
    "confirmationCode": "SONIC-7X9K2M",
    "status": "confirmed",
    "message": "Show this code when you arrive!"
  }
}
Note: Email is required for booking confirmation. Confirmation sent via Resend.

  ]
}
Note: Registration uses email (standard web flow). Welcome email sent via Resend.

💻 Data Models

// OTP Record
interface OTPRecord {
  id: string;
  code: string;
  tableId: string;
  sessionToken: string;
  guestName: string;
  guestEmail?: string;
  bookingId?: string; // null for walk-ins
  createdAt: Date;
  expiresAt: Date;
  used: boolean;
  generatedBy: string; // admin staff ID
}

// Session
interface Session {
  token: string;
  tableId: string;
  tableType: string;
  guestName: string;
  bookingId?: string;
  createdAt: Date;
  expiresAt: Date;
}

// Booking
interface Booking {
  id: string;
  confirmationCode: string;
  tableType: string;
  date: string;
  time: string;
  partySize: number;
  status: "confirmed" | "checked-in" | "completed" | "cancelled";
  guestName: string;
  guestEmail: string;
  occasion?: string;
  specialRequests?: string;
}
